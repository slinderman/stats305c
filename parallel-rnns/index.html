Parallelizing Nonlinear RNNs

Prerequisites

This chapter builds on Recurrent Neural Networks and Deep SSMs and Linear Attention.

Linear RNNs and structured SSMs are fast to train because their recurrences compose as affine functions — a property that admits an O(log⁡T)O(\log T)O(logT)-depth parallel scan. Nonlinear RNNs, like the vanilla RNN and the GRU, do not enjoy this property: each state depends nonlinearly on the previous one, so evaluation seems inherently sequential. This chapter first introduces the parallel scan algorithm, then shows how iterative linearization extends it to nonlinear recurrences — reducing each iteration to a linear dynamical system (LDS) solve that can be parallelized with the same primitive.

The Parallel Scan¶

Consider a linear RNN with time-varying dynamics,

xt=Atxt−1+bt,t=1,…,T,\mbx_t = \mbA_t \mbx_{t-1} + \mbb_t, \qquad t = 1, \ldots, T,xt​=At​xt−1​+bt​,t=1,…,T,
(1)

where xt∈RD\mbx_t \in \reals^Dxt​∈RD, At∈RD×D\mbA_t \in \reals^{D \times D}At​∈RD×D, and bt∈RD\mbb_t \in \reals^Dbt​∈RD. Naively, computing the full trajectory x1:T\mbx_{1:T}x1:T​ requires TTT sequential steps, since each xt\mbx_txt​ depends on xt−1\mbx_{t-1}xt−1​. The parallel scan Blelloch, 1990, also called the associative scan, reduces this to O(log⁡T)O(\log T)O(logT) depth by exploiting the structure of affine composition.

Associativity and Closure¶

Each step of the recurrence is an affine map ft(x)=Atx+btf_t(\mbx) = \mbA_t \mbx + \mbb_tft​(x)=At​x+bt​. Two consecutive steps compose as another affine map:

fj∘fi(x)=Aj(Aix+bi)+bj=(AjAi) x+(Ajbi+bj).f_j \circ f_i(\mbx) = \mbA_j(\mbA_i \mbx + \mbb_i) + \mbb_j = (\mbA_j \mbA_i)\,\mbx + (\mbA_j \mbb_i + \mbb_j).fj​∘fi​(x)=Aj​(Ai​x+bi​)+bj​=(Aj​Ai​)x+(Aj​bi​+bj​).
(2)

Representing each step as a pair (At,bt)(\mbA_t, \mbb_t)(At​,bt​), the composition rule is:

(Aj,bj)⊗(Ai,bi)=(AjAi,  Ajbi+bj).(\mbA_j, \mbb_j) \otimes (\mbA_i, \mbb_i) = (\mbA_j \mbA_i,\; \mbA_j \mbb_i + \mbb_j).(Aj​,bj​)⊗(Ai​,bi​)=(Aj​Ai​,Aj​bi​+bj​).
(3)

This operation is associative — (fk∘fj)∘fi=fk∘(fj∘fi)(f_k \circ f_j) \circ f_i = f_k \circ (f_j \circ f_i)(fk​∘fj​)∘fi​=fk​∘(fj​∘fi​) — and closed — the composition of two affine maps is again an affine map. These two properties are exactly what the parallel scan requires.

Divide-and-Conquer¶

Given TTT affine maps, we want all cumulative compositions ft:1=ft∘⋯∘f1f_{t:1} = f_t \circ \cdots \circ f_1ft:1​=ft​∘⋯∘f1​ for t=1,…,Tt = 1, \ldots, Tt=1,…,T. The parallel scan computes these in two phases:

  • Up-sweep: pair up neighboring maps and reduce in parallel, ⌈log⁡2T⌉\lceil \log_2 T \rceil⌈log2​T⌉ rounds, building products at powers of two.

  • Down-sweep: propagate cumulative products back to fill all positions, another ⌈log⁡2T⌉\lceil \log_2 T \rceil⌈log2​T⌉ rounds.

Together both phases run in O(log⁡T)O(\log T)O(logT) depth with O(T)O(T)O(T) processors, yielding the full trajectory x1:T\mbx_{1:T}x1:T​ in O(T)O(T)O(T) total work — the same as sequential evaluation, but parallelized across the time dimension.

Why diagonal matrices?

Each composition involves a matrix–matrix product costing O(D3)O(D^3)O(D3). Restricting At\mbA_tAt​ to be diagonal reduces this to O(D)O(D)O(D), making the parallel scan practical for large hidden dimensions. This is why S5 and Mamba use diagonal state matrices.

Sequential Evaluation as Root-Finding¶

Let xt∈RD\mbx_t \in \reals^Dxt​∈RD denote the hidden state at time ttt, and let ft+1f_{t+1}ft+1​ denote the (possibly nonlinear, possibly input-dependent) transition function. Sequential evaluation of the recurrence,

xt+1=ft+1(xt),t=0,…,T−1,\mbx_{t+1} = f_{t+1}(\mbx_t), \qquad t = 0, \ldots, T-1,xt+1​=ft+1​(xt​),t=0,…,T−1,
(4)

requires O(T)O(T)O(T) serial steps because xt+1\mbx_{t+1}xt+1​ directly depends on xt\mbx_txt​.

We can reframe the problem: the correct trajectory x1:T∗\mbx_{1:T}^*x1:T∗​ is the unique solution to the system of equations,

xt+1−ft+1(xt)=0,∀t∈{0,…,T−1}.\mbx_{t+1} - f_{t+1}(\mbx_t) = 0, \qquad \forall t \in \{0, \ldots, T-1\}.xt+1​−ft+1​(xt​)=0,∀t∈{0,…,T−1}.
(5)

This is a system of TTT coupled nonlinear equations in TDTDTD unknowns. Fixed-point iteration is a natural approach: start from an initial guess x1:T(0)\mbx_{1:T}^{(0)}x1:T(0)​ (e.g., all zeros) and iteratively refine it by solving a simpler surrogate problem.

A Unifying Framework¶

The key insight of Gonzalez et al. (2026) is that four prominent fixed-point methods — Newton, quasi-Newton, Picard, and Jacobi — all reduce to iterative evaluation of a linear dynamical system. Specifically, each iteration takes the common form,

xt+1(i+1)=ft+1 ⁣(xt(i))+A~t+1 ⁣(xt(i+1)−xt(i)),\mbx_{t+1}^{(i+1)} = f_{t+1}\!\left(\mbx_t^{(i)}\right) + \widetilde{A}_{t+1}\!\left(\mbx_t^{(i+1)} - \mbx_t^{(i)}\right),xt+1(i+1)​=ft+1​(xt(i)​)+At+1​(xt(i+1)​−xt(i)​),
(6)

where A~t+1∈RD×D\widetilde{A}_{t+1} \in \reals^{D \times D}At+1​∈RD×D is an approximate Jacobian of the dynamics. This is a linear recursion in the unknown xt+1(i+1)\mbx_{t+1}^{(i+1)}xt+1(i+1)​, driven by the bias bt+1=ft+1(xt(i))−A~t+1xt(i)\mbb_{t+1} = f_{t+1}(\mbx_t^{(i)}) - \widetilde{A}_{t+1} \mbx_t^{(i)}bt+1​=ft+1​(xt(i)​)−At+1​xt(i)​, which only depends on the previous iterate. Since it is an LDS, it can be evaluated via a parallel scan in O(log⁡T)O(\log T)O(logT) depth.

The four methods differ only in how they choose A~t+1\widetilde{A}_{t+1}At+1​:

MethodA~t+1\widetilde{A}_{t+1}At+1​Cost per iterationParallelization
Newton∂ft+1∂xt(xt(i))\frac{\partial f_{t+1}}{\partial \mbx_t}(\mbx_t^{(i)})∂xt​∂ft+1​​(xt(i)​) — full JacobianO(TD3)O(TD^3)O(TD3)Parallel scan (dense)
Quasi-Newtondiag ⁣[∂ft+1∂xt(xt(i))]\mathrm{diag}\!\left[\frac{\partial f_{t+1}}{\partial \mbx_t}(\mbx_t^{(i)})\right]diag[∂xt​∂ft+1​​(xt(i)​)] — diagonalO(TD)O(TD)O(TD)Parallel scan (elementwise)

All four methods are guaranteed to converge to the correct trajectory x1:T∗\mbx_{1:T}^*x1:T∗​ in at most TTT iterations Gonzalez et al., 2026.

Four Root Finding Methods¶

Newton Iterations¶

Newton’s method for root-finding linearizes the residual r(x1:T)=[x1−f1(x0),…,xT−fT(xT−1)]r(\mbx_{1:T}) = [\mbx_1 - f_1(\mbx_0), \ldots, \mbx_T - f_T(\mbx_{T-1})]r(x1:T​)=[x1​−f1​(x0​),…,xT​−fT​(xT−1​)] using its full Jacobian. Applied to sequential evaluation, each Newton step is the first-order Taylor expansion of the recurrence around the current iterate:

xt+1(i+1)=ft+1 ⁣(xt(i))+∂ft+1∂xt ⁣(xt(i)) ⁣(xt(i+1)−xt(i)).\mbx_{t+1}^{(i+1)} = f_{t+1}\!\left(\mbx_t^{(i)}\right) + \frac{\partial f_{t+1}}{\partial \mbx_t}\!\left(\mbx_t^{(i)}\right)\!\left(\mbx_t^{(i+1)} - \mbx_t^{(i)}\right).xt+1(i+1)​=ft+1​(xt(i)​)+∂xt​∂ft+1​​(xt(i)​)(xt(i+1)​−xt(i)​).
(7)

This is the DEER algorithm of Lim et al. (2024). Because the transition matrix A~t+1\widetilde{A}_{t+1}At+1​ is the full D×DD \times DD×D Jacobian, each iteration requires O(TD2)O(TD^2)O(TD2) memory and O(TD3)O(TD^3)O(TD3) work for the matrix–matrix multiplications in the parallel scan. For large state dimensions, this is prohibitive.

When ft+1f_{t+1}ft+1​ is a linear function of xt\mbx_txt​, the Jacobian is exact and Newton converges in a single iteration — recovering the standard parallel scan for LDSs as a special case.

Quasi-Newton Iterations¶

To reduce the cost of Newton iterations, Gonzalez et al. (2024) replace the full Jacobian with its diagonal:

xt+1(i+1)=ft+1 ⁣(xt(i))+diag ⁣[∂ft+1∂xt ⁣(xt(i))] ⁣(xt(i+1)−xt(i)).\mbx_{t+1}^{(i+1)} = f_{t+1}\!\left(\mbx_t^{(i)}\right) + \mathrm{diag}\!\left[\frac{\partial f_{t+1}}{\partial \mbx_t}\!\left(\mbx_t^{(i)}\right)\right]\!\left(\mbx_t^{(i+1)} - \mbx_t^{(i)}\right).xt+1(i+1)​=ft+1​(xt(i)​)+diag[∂xt​∂ft+1​​(xt(i)​)](xt(i+1)​−xt(i)​).
(8)

This is the ELK (Evaluating Levenberg–Marquardt via Kalman) algorithm. With a diagonal transition matrix, each parallel scan step is an elementwise vector multiplication, reducing cost to O(TD)O(TD)O(TD). The diagonal of the Jacobian can often be computed in closed form for common RNN architectures (e.g., GRUs), or estimated stochastically using the Hutchinson estimator in a single function call.

Picard Iterations¶

Picard iterations set A~t+1=ID\widetilde{A}_{t+1} = I_DAt+1​=ID​, approximating the Jacobian by the identity matrix. The update simplifies to a prefix sum:

xt+1(i+1)=x0+∑s=0tfs+1 ⁣(xs(i))Δ,\mbx_{t+1}^{(i+1)} = \mbx_0 + \sum_{s=0}^{t} f_{s+1}\!\left(\mbx_s^{(i)}\right) \Delta,xt+1(i+1)​=x0​+s=0∑t​fs+1​(xs(i)​)Δ,
(9)

where Δ\DeltaΔ is the discretization step (for ODE-based dynamics). Picard iterations require only vector additions, making them the cheapest per-iteration method. Shih et al. (2023) used them to parallelize sampling in diffusion models.

The identity approximation is faithful when the true Jacobian ∂ft+1/∂xt≈ID\partial f_{t+1}/\partial \mbx_t \approx I_D∂ft+1​/∂xt​≈ID​, which holds for dynamics with small step sizes (e.g., discretized ODEs with fine time steps).

Jacobi Iterations¶

Jacobi iterations set A~t+1=0\widetilde{A}_{t+1} = 0At+1​=0, giving the simplest possible update:

xt+1(i+1)=ft+1 ⁣(xt(i)).\mbx_{t+1}^{(i+1)} = f_{t+1}\!\left(\mbx_t^{(i)}\right).xt+1(i+1)​=ft+1​(xt(i)​).
(10)

Each element of the new trajectory is computed independently — no dependencies between time steps at all. This is embarrassingly parallel: all TTT states can be updated simultaneously in a single kernel call. Song et al. (2021) used Jacobi iterations to accelerate feedforward computation in deep networks.

The zero-Jacobian approximation is faithful when consecutive states evolve nearly independently, i.e., when the true Jacobian ∂ft+1/∂xt≈0\partial f_{t+1}/\partial \mbx_t \approx 0∂ft+1​/∂xt​≈0.

Convergence Analysis¶

Gonzalez et al. (2026) derive a unified convergence bound for all four methods. Let e(i)=x1:T(i)−x1:T∗\mbe^{(i)} = \mbx_{1:T}^{(i)} - \mbx_{1:T}^*e(i)=x1:T(i)​−x1:T∗​ denote the error at iteration iii, and let J~\widetilde{J}J and JJJ denote the block-bidiagonal approximate and true Jacobians of the residual. Then:

∥e(i+1)∥2≤∥J~−1∥2⋅∥J~−J∥2⋅∥e(i)∥2+O ⁣(∥e(i)∥22).\|\mbe^{(i+1)}\|_2 \leq \left\|\widetilde{J}^{-1}\right\|_2 \cdot \left\|\widetilde{J} - J\right\|_2 \cdot \|\mbe^{(i)}\|_2 + O\!\left(\|\mbe^{(i)}\|_2^2\right).∥e(i+1)∥2​≤∥∥​J−1∥∥​2​⋅∥∥​J−J∥∥​2​⋅∥e(i)∥2​+O(∥e(i)∥22​).
(11)

As the error approaches zero, the asymptotic linear convergence rate is:

γ=∥J~−1(J~−J)∥2.\gamma = \left\|\widetilde{J}^{-1}\left(\widetilde{J} - J\right)\right\|_2.γ=∥∥​J−1(J−J)∥∥​2​.
(12)

Convergence is fast (γ≪1\gamma \ll 1γ≪1) when two conditions hold simultaneously:

  1. Small approximation error: ∥J~−J∥2\|\widetilde{J} - J\|_2∥J−J∥2​ is small, i.e., A~t+1\widetilde{A}_{t+1}At+1​ is a faithful approximation of the true Jacobian ∂ft+1/∂xt\partial f_{t+1}/\partial \mbx_t∂ft+1​/∂xt​.

  2. Stable LDS: ∥J~−1∥2\|\widetilde{J}^{-1}\|_2∥J−1∥2​ is small, i.e., the linearized system with transition matrices A~t+1\widetilde{A}_{t+1}At+1​ is stable (spectral norms well below one).

The two conditions trade off against each other:

  • Newton: J~=J\widetilde{J} = JJ=J, so ∥J~−J∥2=0\|\widetilde{J} - J\|_2 = 0∥J−J∥2​=0 and the method converges in one step — but only if the resulting LDS is stable.

  • Quasi-Newton: A~t+1=diag[Jt+1]\widetilde{A}_{t+1} = \mathrm{diag}[J_{t+1}]At+1​=diag[Jt+1​] is typically a better approximation than the identity, and the diagonal LDS is often stable.

  • Picard: A~t+1=ID\widetilde{A}_{t+1} = I_DAt+1​=ID​, which is faithful when ∂ft+1/∂xt≈ID\partial f_{t+1}/\partial \mbx_t \approx I_D∂ft+1​/∂xt​≈ID​ but the resulting LDS has unit eigenvalues, making ∥J~−1∥2=O(T)\|\widetilde{J}^{-1}\|_2 = O(T)∥J−1∥2​=O(T) — slow for long sequences.

  • Jacobi: A~t+1=0\widetilde{A}_{t+1} = 0At+1​=0, so ∥J~−1∥2=1\|\widetilde{J}^{-1}\|_2 = 1∥J−1∥2​=1 (minimal), but the approximation error ∥J~−J∥2=∥J∥2\|\widetilde{J} - J\|_2 = \|J\|_2∥J−J∥2​=∥J∥2​ may be large.

Practical guidance

Use the simplest approximate Jacobian possible, but no simpler. The goal is to find the cheapest A~t+1\widetilde{A}_{t+1}At+1​ such that (a) A~t+1≈∂ft+1/∂xt\widetilde{A}_{t+1} \approx \partial f_{t+1}/\partial \mbx_tAt+1​≈∂ft+1​/∂xt​ and (b) the induced LDS is stable. Checking these conditions requires understanding the dynamics of the specific model.

Conclusion¶

The parallel scan turns any associative, closed operation into an O(log⁡T)O(\log T)O(logT)-depth computation. For nonlinear RNNs, iterative linearization reduces each fixed-point iteration to an LDS — making the parallel scan applicable even when the original dynamics are nonlinear. All four methods are guaranteed to converge in at most TTT iterations; the rate depends on how faithfully A~t+1\widetilde{A}_{t+1}At+1​ approximates the true Jacobian and on the stability of the induced LDS.
