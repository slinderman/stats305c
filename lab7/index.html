Part 3: Autoregressive HMMs¶

Autoregressive hidden Markov models (ARHMMs) replace the Gaussian observations with an AR model:

p(x,z∣Θ)=Cat(z1∣π)∏t=2TCat(zt∣Pzt−1)∏t=1Tp(xt∣x1:t−1,zt)\begin{align} p(x, z \mid \Theta) &= \mathrm{Cat}(z_1 \mid \pi) \prod_{t=2}^{T} \mathrm{Cat}(z_t \mid P_{z_{t-1}}) \prod_{t=1}^T p(x_t \mid x_{1:t-1}, z_t) \end{align}p(x,z∣Θ)​=Cat(z1​∣π)t=2∏T​Cat(zt​∣Pzt−1​​)t=1∏T​p(xt​∣x1:t−1​,zt​)​​
(2)

The model is “autoregressive” because xtx_txt​ depends not only on ztz_tzt​ but on x1:t−1x_{1:t-1}x1:t−1​ as well. The precise form of this dependence varies; here we will consider linear Gaussian dependencies on only the most recent LLL timesteps,:

p(xt∣x1:t−1,zt)=N(xt∣∑l=1LAzt,lxt−l+bzt,Qzt)for t>L\begin{align} p(x_t \mid x_{1:t-1}, z_t) &= \mathcal{N}\left(x_t \mid \sum_{l=1}^L A_{z_t,l} x_{t-l} + b_{z_t}, Q_{z_t} \right) \qquad \text{for } t > L \end{align}p(xt​∣x1:t−1​,zt​)​=N(xt​∣l=1∑L​Azt​,l​xt−l​+bzt​​,Qzt​​)for t>L​​
(3)

To complete the model, assume

p(xt∣x1:t−1,zt)=N(xt∣0,I)for t≤L\begin{align} p(x_t \mid x_{1:t-1}, z_t) &= \mathcal{N}\left(x_t \mid 0, I \right) \qquad \text{for } t \leq L \end{align}p(xt​∣x1:t−1​,zt​)​=N(xt​∣0,I)for t≤L​​
(4)

The new parameters are Θ=π,P,{{Ak,l}l=1L,bk,Qk}k=1K\Theta = \pi, P, \{\{A_{k,l}\}_{l=1}^L, b_{k}, Q_k\}_{k=1}^KΘ=π,P,{{Ak,l​}l=1L​,bk​,Qk​}k=1K​, which include weights Ak,l∈RD×DA_{k,l} \in \mathbb{R}^{D \times D}Ak,l​∈RD×D for each of the KKK states and the LLL lags, and a bias vector bk∈RDb_k \in \mathbb{R}^Dbk​∈RD.

Note that we can write this as a simple linear regression,

p(xt∣x1:t−1,zt)=N(xt∣Wkϕt,Qzt)\begin{align} p(x_t \mid x_{1:t-1}, z_t) &= \mathcal{N}\left(x_t \mid W_k \phi_t , Q_{z_t} \right) \end{align}p(xt​∣x1:t−1​,zt​)​=N(xt​∣Wk​ϕt​,Qzt​​)​​
(5)

where ϕt=(xt−1,…,xt−L,1)∈RLD+1\phi_t = (x_{t-1}, \ldots, x_{t-L}, 1) \in \mathbb{R}^{LD +1}ϕt​=(xt−1​,…,xt−L​,1)∈RLD+1 is a vector of covariates (aka features) that includes the past LLL time steps along with a 1 for the bias term.

Wk=[Ak,1Ak,2…AkLbk]∈RD×LD+1\begin{align} W_k = \begin{bmatrix} A_{k,1} & A_{k,2} & \ldots & A_{kL} & b_k \end{bmatrix} \in \mathbb{R}^{D \times LD + 1} \end{align}Wk​=[Ak,1​​Ak,2​​…​AkL​​bk​​]∈RD×LD+1​​
(6)

is a block matrix of the autoregressive weights and the bias.

Note that the covariates are fixed functions of the data so we can precompute them, if we know the number of lags LLL.

Part 4: Fit the ARHMM to mouse videos¶

Now we’ll load in some real data from depth video recordings of freely moving mice. This data is from the Datta Lab at Harvard Medical School. The references are given at the top of this notebook.

The video frames, even after cropping, are still 80x80 pixels. That’s a 3600 dimensional observation. In practice, the frames can be adequately reconstructed with far fewer principal components. As little as ten PCs does a pretty good job of capturing the mouse’s posture.

The Datta lab has already computed the principal components and included them in the NWB. We’ll extract them, along with other relevant information like the centroid position and heading angle of the mouse, which we’ll use for making “crowd” movies below. Finally, they also included labels from MoSeq, an autoregressive (AR) HMM. You’ll build an ARHMM in Part 3 of the lab and infer similar discrete latent state sequences yourself!

Plot the state usage histogram¶

The state usage histogram shows how often each discrete state was used under the posterior distribution.
